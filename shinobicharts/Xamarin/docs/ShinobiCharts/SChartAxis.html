SChartAxis Class

Abstract base class for a horizontal or vertical axis providing a visual scale for chart data

Syntax

[Foundation.Register("SChartAxis", true)]
public abstract class SChartAxis : Foundation.NSObject

See Also

SChartNumberAxis
SChartDateTimeAxis
SChartDiscontinuousNumberAxis
SChartDiscontinuousDateTimeAxis
SChartCategoryAxis
SChartLogarithmicAxis
SChartAxisStyle

Remarks

The SChartAxis is responsible for managing the coordinate space of the chart. It is the link between the set of real data in a series and the laying down of that series on a chart in a desired manner. Each series on the chart is linked to an axis and this SChartAxis maps the data in each of it's linked series to pixel values.

For cartesian series, it is expected that you will provide a subclass of SChartAxis each for the XAxis and YAxis of the chart. If not set, both axes will default to instances of ShinobiCharts.SChartNumberAxis.

The axis range can be set to a desired range or left to calculate its own minimum and maximum. When auto-calculating the range, it will consider all data series associated with it. You can set the desired range by passing in an ShinobiCharts.SChartRange object when constructing the axis, or by setting the SChartAxis.DefaultRange property.

The axis is also the home of the tick marks and their labels. These can be auto-calculated or set to specific values.

More information about using SChartAxis can be found in the user guide.

Documentation for this section has not yet been entered.

Remarks

For example, if your AxisRange is 1 to 9, your AnchorPoint is 0 and the CurrentMajorTickFrequency is 2, you will get tickmarks and labels at 2, 4, 6 and 8, assuming that the labels' text and the clipping settings permit.

Regardless of whether a tick mark frequency has been set or automatically calculated, it must start somewhere. This value acts as the origin point for tickmarks on the axis.

Remarks

Axis objects are universal and may be used as an x-axis (horizontal) or as a y-axis (vertical).

  • SChartOrientation.Horizontal: Configures the axis to be used as an x-axis (horizontal).
  • SChartOrientation.Vertical: Configures the axis to be used a y-axis (vertical).

CalculateMajorTickFrequencyFromDiscontinuousRange Property

This property controls how the auto-tick-frequency calculations are done.

Syntax

[get: Foundation.Export("calculateMajorTickFrequencyFromDiscontinuousRange")]
[set: Foundation.Export("setCalculateMajorTickFrequencyFromDiscontinuousRange:")]
public virtual bool CalculateMajorTickFrequencyFromDiscontinuousRange { get; set; }

Value

Documentation for this section has not yet been entered.

Remarks

Normally, the axis will use the full range of data to calculate how often to place a tick-mark, however if the range has discontinuities, this option allows the axis to exclude the discontinuities in its calculations of the range over which the tick marks are added.

For example - you have a year range, but are excluding 7 months using discontinuous periods. If this property is false, the tick labels will be added as if the range is one year. If this property is true, the tick labels will be added as if the range is 5 months (12 - 7).

If true, the axis will exclude discontinuities when calculating the range.

By defaults, this property is false.

SetRange Method

Attempts to set the current visible range SChartAxis.AxisRange to a range with the given minimum and maximum values.

Syntax

[Foundation.Export("setRangeWithMinimum:andMaximum:")]
public virtual bool SetRange (Foundation.NSObject minimum, Foundation.NSObject maximum)

Parameters

minimum
The minimum value to be displayed, in data terms.
maximum
The maximum value to be displayed, in data terms.

Returns

Whether or not the operation was successful.

Remarks

Given any restrictions on setting the range, such as SChartAxis.AllowPanningOutOfMaxRange etc, this method will attempt to set the current axis range.

The permissable types of minimum and maximum will vary depending on the type of axis in use. The range of an SChartNumberAxis should be set using two objects of type NSNumber for Minimum and Maximum, whilst that of an SChartDateTimeAxis can be configured using either NSNumber or NSDate minima and maxima. In the case of SChartCategoryAxis, the first value has a nominal integer value of '0' and the nth value, 'n-1'. A few examples:
Example
// Range from 20 to 140 on an SChartNumberAxis.
myNumberAxis.SetRange(new NSNumber(20), new NSNumber(140));

// Range from June 2013 to January 2014 (approx.) on an SChartDateTimeAxis.
myDateTimeAxis.SetRange(NSDate.FromTimeIntervalSince1970(86400.0*365*43.5), NSDate.FromTimeIntervalSince1970(86400.0*365*44));

// Range between the third and fifth elements on an SChartCategoryAxis.
myCategoryAxis.SetRange(new NSNumber(2), new NSNumber(6));

Value

Documentation for this section has not yet been entered.

Remarks

If SChartAxis.EnableMomentumZooming is set to true, the velocity of the zoom pinch gesture will decay over a number of increments. These increments are a fixed time period specified here. During this fixed period the velocity will decay by a factor SChartAxis.ZoomMomentumFactor.
